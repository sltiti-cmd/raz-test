# Claude Code Handoff: Stacey老师 RAZ升级测试 UI

请按 Claude Design 里的设计稿实现/改造「Stacey老师 RAZ升级测试」网页应用。

Claude Design 链接：
https://claude.ai/design/p/019e2b9a-6571-7c0f-9f0e-92fccf298222?file=index.html

## 目标

把现有网页改成一个面向小学生和家长的 RAZ 分级阅读升级测试系统。整体要有“高级可爱”的亲和感，同时保持“专业测评系统”的可信感。

品牌名称：
Stacey老师 RAZ升级测试

不要出现：
- 小麦亲子英语 logo
- 过于幼稚的卡通风
- 太商业化、培训机构感太重的设计
- 大面积高饱和颜色

## 视觉方向

风格关键词：
高级可爱、温暖、清爽、专业、亲和、低龄友好、教育感、轻量化测评、奶油色背景、圆角卡片、柔和阴影、清晰排版、适合家长截图分享。

推荐设计语言：
- 背景：浅奶油色、米白色、淡暖色
- 主色：柔和薄荷绿、湖水蓝、奶油黄、浅橙色
- 强调色：低饱和，用于按钮、进度条、选中状态
- 卡片：大圆角、轻微阴影、柔和边框
- 字体：清晰现代，适合儿童阅读
- 英文文章和题目：字号足够大，行距舒适
- 图标：少量书本、星星、耳机、铅笔、奖章即可，不要过花

字体建议：
- 中文：Noto Sans SC 或系统中文字体
- 英文/界面：Nunito 或圆润清晰的 sans-serif
- 报告页可少量使用 Lora 风格的 serif 做证书感标题
- 数字、题号、进度可用 JetBrains Mono 或等宽风格

## 需要实现的页面

### 1. 首页大厅 Home

目标：让用户一眼知道这是一个分级阅读测试入口。

内容：
- 顶部标题：Stacey老师 RAZ升级测试
- 副标题：选择级别，完成测试后自动生成阅读诊断报告
- 级别卡片：
  - A级别 已开放
  - D级别 敬请期待
  - G级别 敬请期待
  - H级别 敬请期待
  - K级别 敬请期待
  - O级别 敬请期待
- A级别卡片需要有“开始测试”按钮
- 未开放级别要显示锁定/敬请期待状态

### 2. 测试页 Test

桌面端：
- 双栏或三栏结构
- 左侧题号导航 1-20
- 中间阅读文章区域
- 右侧/下方题目和选项
- 顶部显示当前级别、进度、剩余/完成状态
- 选项要有默认、hover、选中、正确、错误状态
- 提交时出现确认弹窗

移动端：
- 单列布局
- 题号横向滚动
- 文章和题目上下排列
- 按钮足够大，适合触屏

交互：
- 可以点击选项
- 可以切换题号
- 可以提交测试
- 提交后跳转到报告页

### 3. 结果报告页 Report

做成适合家长截图分享的 A4/报告卡风格。

内容：
- 学生/测试信息
- 总分
- 正确率
- 错题清单
- 阅读能力分析
- 文体表现
- 提升建议
- 温暖但专业的结语

视觉：
- 卡片像“阅读诊断报告”
- 保持奶油底、柔和边框、专业排版
- 不要像普通后台表格

### 4. 后台统计 Admin

内容：
- 概览卡片：总测试人数、平均分、完成率等
- 分数分布
- 学生记录表
- 搜索/筛选/排序
- 导出 CSV 按钮

风格：
- 比首页更工作台一点，但仍保持品牌一致
- 信息密度适中，不要拥挤

### 5. 交互状态

请补齐这些状态：
- 按钮：默认、hover、disabled、loading
- 选项：默认、hover、selected、correct、wrong
- 题号：未做、已做、当前、错题
- 弹窗：提交确认、完成提示
- Toast：保存成功、导出成功

## 实现要求

请优先复用当前项目已有技术栈。如果是 React/Vite/Tailwind 项目，就用 React + Tailwind 实现；如果项目不是 Tailwind，请尽量用现有 CSS 体系实现相同视觉。

请不要只做静态截图，要做成可点击的网页界面：
- 首页能进入测试页
- 测试页能选择答案和切换题号
- 提交后能进入报告页
- 后台筛选和导出按钮至少有前端交互反馈

请同时检查桌面和手机尺寸，保证文字不溢出、按钮不挤压、卡片不重叠。

- 还有哪些地方需要我人工确认
