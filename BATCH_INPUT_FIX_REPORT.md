# 批量输入答案按钮修复报告

## 修改文件

- `src/components/BatchInput.jsx`
- `src/components/BatchInputModal.jsx`
- `src/pages/TestPage.jsx`
- `src/pages/PlacementPage.jsx`

## 原因

原来的批量输入区域是题目栏下方的内嵌展开块，顶部按钮只是切换同一个页面内的 `showBatch`。在手机端布局中，这个展开区域可能位于当前视口之外，点击顶部按钮后用户看不到任何变化，因此表现为“无反应”。

## 修复方式

- 新增统一弹窗组件 `BatchInputModal`。
- 升级测试页和插班测试页都使用同一个弹窗入口。
- 顶部按钮和下方按钮都调用同一个 `openBatchInput`。
- `openBatchInput` 使用 `preventDefault` 和 `stopPropagation` 后直接打开弹窗。
- 批量输入成功后关闭同一个弹窗，并保留原来的答案解析和填入逻辑。

## 手机端结果

- 手机尺寸下，顶部【批量输入答案】按钮点击后立即打开弹窗。
- 弹窗使用固定定位和更高层级，不依赖下方区域是否可见。
- 输入框、批量填入按钮、关闭按钮均可点击。

## 功能保留

- 支持 `A B C D`
- 支持 `A,B,C,D`
- 支持 `ABCD`
- 支持每5题换行
- 支持空格分隔
- 输入20题答案后可自动填入全部题目。

## 验收结果

- A级测试页：顶部批量输入按钮可打开弹窗。
- C级插班测试页：顶部批量输入按钮可打开弹窗。
- H级插班测试页：顶部批量输入按钮可打开弹窗。
- 手机尺寸：顶部批量输入按钮可打开弹窗。
- 桌面端：顶部和下方批量输入按钮均打开同一个弹窗。
- A级正确答案批量填入后，可正常提交并生成 100/100 报告。
